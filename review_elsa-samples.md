# Elsa Samples 주요 코드 리뷰 (Top 10)

### 1. HelloWorld.cs
- **기능**: Elsa 엔진을 초기화하고 가장 단순한 형태의 콘솔 기반 워크플로를 실행하는 기초 예제입니다.
- **디자인 패턴**: **Facade Pattern**, **Simple Factory Pattern**.
- **중요성**: 엔진의 기본 구조와 실행 사이클을 이해하기 위한 가장 중요한 학습 포인트입니다.
- **상세 설명**: `WriteLine` 활동을 사용하여 메시지를 출력하는 워크플로를 정의하며, `WorkflowRunner`를 통해 동기적으로 실행하는 과정을 보여줍니다. 서비스 컨테이너 설정 및 활동 인스턴스화의 기본 원리를 제시합니다.

### 2. HttpWorkflow.cs
- **기능**: HTTP 엔드포인트를 열고 요청을 수신하여 조건에 따라 응답하는 웹 기반 워크플로 시나리오입니다.
- **디자인 패턴**: **Adapter Pattern**, **Middleware Pattern**.
- **중요성**: Elsa를 웹후크(Webhook) 수신기나 마이크로서비스 컨트롤러로 활용하는 방법을 제시합니다.
- **상세 설명**: `HttpEndpoint` 트리거와 `WriteHttpResponse` 활동을 연결하며, 쿼리 파라미터나 요청 본문에서 변수를 추출하여 로직에 활용하는 동적 데이터 처리 방법을 설명합니다.

### 3. UserTaskExample.razor
- **기능**: 워크플로 도중 사람의 승인이 필요한 'User Task'를 구현하고, 이를 Blazor UI와 연동하는 예제입니다.
- **디자인 패턴**: **Observer Pattern**, **State Machine Pattern**.
- **중요성**: 장기 실행 워크플로(Long-running workflows)의 핵심인 대기 및 재개 메커니즘을 보여줍니다.
- **상세 설명**: 워크플로가 사용자의 입력을 기다리며 중단(Suspend)되는 과정과, UI에서 버튼 클릭 시 북마크를 깨워 실행을 재개하는 로직을 단계별로 구현합니다.

### 4. ErrorHandlingWorkflow.cs
- **기능**: 활동 실행 도중 발생하는 예외를 포착하고, 보상 트랜잭션이나 재시도 로직을 수행하는 오류 처리 예제입니다.
- **디자인 패턴**: **Try-Catch-Finally Pattern**, **Circuit Breaker Pattern** (개념적).
- **중요성**: 프로덕션 환경에서 워크플로의 견고성을 확보하기 위한 필수적인 설계 기법을 제시합니다.
- **상세 설명**: `Faulted` 상태의 워크플로를 분석하고, 특정 오류 코드에 따라 알림을 보내거나 데이터 롤백 활동을 수행하는 흐름 제어 방법을 설명합니다.

### 5. ParallelWorkflow.cs
- **기능**: 여러 개의 활동을 동시에(병렬) 실행하고, 모든 작업이 완료될 때까지 기다렸다가 다음 단계로 넘어가는 예제입니다.
- **디자인 패턴**: **Fork-Join Pattern**, **Composite Pattern**.
- **중요성**: 복잡한 비즈니스 프로세스에서 시간 효율성을 극대화하는 병렬 처리 기법을 보여줍니다.
- **상세 설명**: `Parallel` 노드를 사용하여 실행 흐름을 여러 갈래로 나누고, `Join` 노드를 통해 모든 분기가 합쳐지는 동기화 메커니즘을 코드와 그래프 구조로 설명합니다.

### 6. CustomActivity.cs
- **기능**: Elsa의 표준 인터페이스를 상속받아 고유한 비즈니스 로직을 담은 사용자 정의 활동을 개발하는 방법입니다.
- **디자인 패턴**: **Template Method Pattern**, **Strategy Pattern**.
- **중요성**: 엔진의 기능을 무한히 확장할 수 있는 핵심 개발 방법론을 제시합니다.
- **상세 설명**: `Activity` 기본 클래스를 상속받아 입력/출력 속성을 정의하고, `OnExecuteAsync` 메서드 내에서 외부 라이브러리 연동 및 결과값을 반환하는 상세 과정을 포함합니다.

### 7. RecurringWorkflow.cs
- **기능**: 크론(Cron) 표현식을 사용하여 특정 시간 간격으로 자동으로 실행되는 주기적 작업 워크플로 예제입니다.
- **디자인 패턴**: **Scheduler Pattern**, **Cron Job Pattern**.
- **중요성**: 시스템 자동화, 주기적 데이터 수집, 보고서 발송 등 배치 작업 구현의 표준을 보여줍니다.
- **상세 설명**: `Cron` 트리거를 설정하고 Quartz.NET 또는 기본 타이머 서비스와 연동되어 백그라운드에서 워크플로 인스턴스가 생성되는 주기를 관리하는 방법을 설명합니다.

### 8. DependencyInjectionWorkflow.cs
- **기능**: 워크플로 내부에서 외부에서 주입된 .NET 서비스(DB Context, API Client 등)를 사용하는 방법을 제시합니다.
- **디자인 패턴**: **Dependency Injection Pattern**, **Service Locator Pattern** (활동 내부적 사용).
- **중요성**: 워크플로 로직과 외부 시스템 간의 결합도를 낮추고 테스트 가능성을 높이는 설계 기법입니다.
- **상세 설명**: 활동 클래스의 생성자나 `Expression`을 통해 DI 컨테이너에 등록된 서비스를 가져와 실제 비즈니스 로직에 활용하는 모범 사례를 제공합니다.

### 9. PersistenceConfiguration.cs
- **기능**: 워크플로 상태를 저장하기 위해 SQLite, SQL Server 등 다양한 데이터베이스 저장소를 전환하여 설정하는 예제입니다.
- **디자인 패턴**: **Strategy Pattern**, **Adapter Pattern**.
- **중요성**: 프로젝트 환경에 맞춰 유연하게 영속성 레이어를 선택하고 구성하는 방법을 이해하게 합니다.
- **상세 설명**: `AddElsa` 설정 시점에 `UsePersistence` 옵션을 통해 서로 다른 Provider를 연결하고 연결 문자열을 관리하는 구성 코드를 보여줍니다.

### 10. SignalWorkflow.cs
- **기능**: 외부 시스템에서 보내는 임의의 신호(Signal)를 수신하여 대기 중인 워크플로를 깨우는 시그널링 예제입니다.
- **디자인 패턴**: **Observer Pattern**, **Event-Driven Pattern**.
- **중요성**: 비정형 이벤트나 사용자 작업 흐름에 따라 워크플로의 방향을 결정하는 실시간 상호작용 기법입니다.
- **상세 설명**: `SignalReceived` 활동을 배치하여 특정 토큰을 대기시키고, 외부 API에서 해당 토큰과 함께 신호를 보낼 때 북마크가 매칭되어 워크플로가 재개되는 전체 과정을 구현합니다.
